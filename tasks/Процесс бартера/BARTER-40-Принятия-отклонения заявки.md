# BARTER-40: Принятия/отклонения заявки

## Описание задачи
Реализация функционала для принятия или отклонения заявок в процессе бартера.

## Список действий
1. Реализовать endpoint для принятия заявки
2. Реализовать endpoint для отклонения заявки
3. Обновить статус заявки при принятии/отклонении
4. Обеспечить права доступа для данной операции
5. Уведомить участников о принятии/отклонении заявки
6. Обновить статусы связанных сущностей при принятии
7. Залогировать действия по принятию/отклонению
8. Протестировать оба сценария
