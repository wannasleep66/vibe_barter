# BARTER-39: Отклонение заявок после 72 часов простоя

## Описание задачи
Реализация автоматического отклонения заявок, которые находятся в ожидании более 72 часов.

## Список действий
1. Реализовать фоновый процесс проверки старых заявок
2. Определить критерии "простоя" заявки (без активности)
3. Реализовать автоматическое изменение статуса на "отклонена"
4. Уведомить пользователей об автоматическом отклонении
5. Залогировать причину автоматического отклонения
6. Обеспечить настройку времени ожидания (72 часа)
7. Тестировать работу механизма с разными сценариями
8. Обеспечить производительность при большом количестве заявок
